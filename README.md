Personal dotfiles for Ubuntu 24.04 LTS, managed declaratively with the
flake-based [Nix home-manager](https://github.com/nix-community/home-manager).

## Layout

```
install.sh        # one-shot provisioning: apt repos/packages, rust, nix, home-manager
flake.nix         # flake inputs + per-host homeConfigurations + systemConfigs
home.nix          # base config: packages, bash, git, and the dotfiles (home.file)
system.nix        # system-level config (/etc, systemd) via numtide/system-manager
hosts/            # per-host modules, selected as will@<hostname>
modules/gnome/    # dconf settings (generated by gnome-settings-export)
config/           # raw config files home-manager symlinks into $HOME
├── wezterm/      #   .wezterm.lua
└── tmux/         #   .tmux.conf
bin/              # helper scripts (gnome-settings-export, update-config.sh)
packages/         # apt manifest (apt.txt)
apparmor/         # AppArmor profiles for nix GUI-app sandboxes
```

## Install

```bash
git clone https://github.com/willcforte/.dotfiles.git ~/.dotfiles
cd ~/.dotfiles
./install.sh          # apt + rust + nix + home-manager (asks for sudo)
```

## Applying changes

After editing any `.nix` file, re-apply the configuration:

```bash
update-config         # home-manager switch, then system-manager switch (prompts for sudo)
```

`update-config` applies both the user config (home-manager) and the system
config (`system.nix` via system-manager). The system step self-escalates and
will prompt for your sudo password.

Frequently-edited dotfiles (`.wezterm.lua`, `.tmux.conf`, `~/.claude/settings.json`)
are symlinked back to this repo via `mkOutOfStoreSymlink`, so edits to them take
effect immediately without a rebuild.
